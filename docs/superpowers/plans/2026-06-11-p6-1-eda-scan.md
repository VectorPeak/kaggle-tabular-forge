# P6.1 EDA Scan Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 `kaggle-tabular-forge` 增加可执行的 P6.1 EDA 扫描入口，能够从 YAML 配置读取比赛数据并产出 `eda_summary.md`、`eda_manifest.json`、`leakage_watchlist.json`、`feature_backlog.json`。

**Architecture:** 沿用仓库现有模式，CLI 只做参数接线，配置通过 schema 校验与 typed loader 解析，EDA 逻辑落在 `src/ktabforge/eda/`，artifact 直接写入 `artifacts/eda_findings/<competition>/<eda_id>/`。第一版只实现静态扫描与风险提示，不做复杂特征构建和 registry 回填。

**Tech Stack:** Python 3.12, Typer, pandas, jsonschema, PyYAML, pytest, ruff
